## Goal

- Fix all sharing flow issues (8 root causes identified) — unique constraint errors, missing doctor info, audit logging, mark-as-read UX, missing QR/AI on all doctor pages, real-time badges, post-login redirect

## Constraints & Preferences

- Production schema changes require user approval before apply
- Fixes must be additive where possible to avoid breaking existing flows
- Audit trail required for both push-based and pull-based sharing views
- Doctor landing page must work even when `doctor_profiles` row is missing
- QR FAB + AI Assistant must appear on all doctor dashboard pages (home, patients, profile)
- Card click in SharedWithMeSection must not auto-mark as viewed — explicit "Mark Seen" action

## Key Decisions

- Used `upsert` with `ignoreDuplicates` for share action instead of `ON CONFLICT DO NOTHING` raw SQL — same effect, cleaner Supabase SDK
- SharedWithMeSection: separated "View Records" (navigation) from "Mark Seen" (explicit toggle) — no auto-marking
- `markSharedReportViewed` fetches share row first to get patient_id/report_id for audit log — one extra query but ensures data integrity
- Created fix migration file `20260612000001` for any env that already has the old FK — forward compatibility
- `DoctorPageShell` fetches own Profile + DoctorProfile; passes `recentPatients: []` to AI (empty) — avoids server/client data passing
- Profile page keeps own QR card in hero + gets FAB from shell — both coexist without conflict
